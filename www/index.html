Aus aktuellem Anlass: Ganz offensichtlich finden eine Menge Leute hier her, die die S.O.S-Sendung am Mittwoch auf Pro7 gesehen haben. Dazu ein paar Anmerkungen: Obwohl mein "Rasterizer" den selben Zweck erfüllt, ist dies nicht die Software, die die SOS-Fachfrau verwendet hat. Was Ihr sucht, ist der Rasterbator.

Natürlich freue ich mich dennoch über Euren Besuch hier und vielleicht hat ja doch der ein oder andere Lust darauf, meinen Rasterizer einmal auszuprobieren.

Dennoch, ein Wort der Warnung (ich bekomme zur Zeit die ein- oder andere Mail dazu): Mein Tool befindet sich in einem sehr frühen Entwicklungsstadium. Es kann zwar etwas mehr als der Rasterbator, ist aber auch nicht so einfach zu bedienen und vor allem nicht sehr einfach in Gang zu bekommen.

Eine ausführliche Anleitung (in der ersten Version mit vermutlich reichlich Rechtschreibfehlern) findet ihr HIER

Langer Rede kurzer Sinn: Wenn Ihr von S.O.S. kommt, sucht Ihr vermutlich den Rasterbator.

Nachtrag: Ich habe inzwischen mitbekommen, das der Rasterbator wohl hoffnungslos überlastet ist, was angesichts einer Nennung in einem TV-magazin wohl auch nicht verwunderlich ist. Genau aus solchen Gründen wollte ich ein Tool bauen, das auch ohne Internet funktioniert.

Nachtrag [2]: Wenn alles gut läuft, setze ich heute noch ein Forum für den Rasterizer auf und es gibt eine neue Version vom Rasterizer, die sich dann direkt starten lässt. Drückt die Daumen :)

Rasterizer at a glance

  • Kind of a commandline version of the
    • Rasterbator
  • Current Version (CLI version): 0.5
  • Current Version (GUI version): EXPERIMENTAL 0.2
  • Still limited feature set:
    • Only one resizing parameter: # of horizontal pages
  • Apart from that, it's already pretty usable. See below for examples.
  • Released under the GPL.

Changelog

Experimental GUI 0.2

  • NEW: Implemented FileFilters for FileSelectors.
  • NEW: Implemented Locale support for GUI strings (default: en, and de)

Special Release "Agnes": Experimental GUI 0.1

  • NEW: Experimental GUI Release. Download rasterizergui_0_1.zip!
  • CHG: Totally restructured the code, (not finished), to allow for easy GUI creation. Output will be totally different in the cli version.

0.5

  • NEW: -c is now switching experimental color support (the rasterbator way)
  • CHG: -m switches cropmarks (same as -c in 0.4)

0.4

  • NEW: -c prints cropmarks (intelligently, only on the "insides")
  • NEW:-call prints all cropmarks
  • BUGFIX: Some strange errors in calculation of last dot row.

0.3

  • NEW: -l functional
  • NEW: -s (page Size) implemented (takes the following parameters: A4, A3, LETTER, LEGAL)
  • NEW: -d (dot Size) implemented (takes values of Point (10pt is default)
  • NEW: automatic calculation of margins etc.

0.2

  • Code cleanup (one shouldn't write such software between 1 and 4 AM :)

0.1

  • Inital proof of concept

Planned Features for 1.0

  • Parameter to allow for landscape printing (done)
  • Parameters to allow other paper formats (A3, Letter, whatever) (done for A4, A3, LETTER and LEGAL)
  • Automatic margins (done)
  • Parameters for other raster methods (e.g. squares)
  • Additional information on last page, signature
  • Parameter to switch on crop marks for fine creations without gaps (done!)
  • Parameter to specify vertical # of pages
  • Parameter to specify dot size (done)
